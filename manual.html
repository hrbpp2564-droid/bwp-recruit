1. ภาพรวมระบบ

1.1 จุดประสงค์

BWP Recruitment Management System เป็นระบบบริหารงานสรรหาบุคลากรแบบครบวงจร ครอบคลุมตั้งแต่การวางแผนอัตรากำลัง การเปิดคำขอตำแหน่ง การคัดกรองผู้สมัคร การนัดสัมภาษณ์ การจัดทำ Offer จนถึงกระบวนการ Onboarding พนักงานใหม่

1.2 คุณสมบัติหลัก

  • Dashboard — แสดงภาพรวมข้อมูลสรรหาแบบ Real-time (3 รูปแบบ Layout)
  • Manpower Planning — วางแผนอัตรากำลังแต่ละแผนก (แผน vs จริง)
  • Requisition Workflow — เปิดคำขอพร้อมขั้นตอนอนุมัติ 4 ระดับ
  • Recruitment Pipeline — ติดตามผู้สมัครแบบ Kanban 8 สเตจ
  • Interview Management — นัดสัมภาษณ์ + ประเมินคะแนน 6 หัวข้อ
  • Offer Management — จัดทำและอนุมัติ Offer พร้อม Approval Flow
  • Onboarding — Checklist งาน Onboard พนักงานใหม่
  • AI Analytics — วิเคราะห์เชิงลึกด้วย AI (Matching, Screening, Insights)
  • RBAC — ระบบสิทธิ์ตามบทบาท 6 ระดับ, 14 โมดูล
  • Multi-channel Notifications — แจ้งเตือนผ่าน LINE และ Email

1.3 ความต้องการระบบ

รายการข้อกำหนด
เว็บเบราว์เซอร์Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
ความละเอียดหน้าจอแนะนำ 1366×768 ขึ้นไป (รองรับ Responsive)
อินเทอร์เน็ตความเร็วขั้นต่ำ 5 Mbps

2. การเข้าสู่ระบบ (Login)

2.1 ขั้นตอนการเข้าสู่ระบบ

1 เปิดเว็บเบราว์เซอร์ แล้วไปที่ URL ของระบบ
2 กรอก ชื่อผู้ใช้ (Username) และ รหัสผ่าน (Password)
3 คลิกปุ่ม "เข้าสู่ระบบ"
4 ระบบจะพาเข้าสู่หน้า Dashboard ตามสิทธิ์ของบทบาทที่ได้รับ
หมายเหตุ: Session จะหมดอายุหลัง 24 ชั่วโมง — ต้องเข้าสู่ระบบใหม่

2.2 การเปลี่ยนรหัสผ่าน

  1. คลิกที่ชื่อผู้ใช้ (มุมขวาบน) หรือไปที่เมนู ตั้งค่า
  2. เลือก "เปลี่ยนรหัสผ่าน"
  3. กรอกรหัสผ่านเดิม แล้วตั้งรหัสผ่านใหม่ (ขั้นต่ำ 8 ตัวอักษร)
  4. คลิก "บันทึก"
ข้อควรระวัง: หากลืมรหัสผ่าน ให้แจ้ง Super Admin เพื่อรีเซ็ตรหัสผ่านให้

2.3 การออกจากระบบ (Logout)

คลิกปุ่ม "ออกจากระบบ" ที่แถบด้านซ้ายมือล่างสุด ระบบจะลบ Session และกลับไปหน้า Login

4. แดชบอร์ด (Dashboard)

4.1 ภาพรวม

หน้า Dashboard แสดงข้อมูลสำคัญทั้งหมดในมุมมองเดียว เพื่อให้ผู้บริหารและทีม HR ติดตามสถานะงานสรรหาได้ทันที

4.2 ข้อมูลสรุป (KPI Cards)

  • พนักงานปัจจุบัน — จำนวนพนักงานทั้งหมดในองค์กร
  • ตำแหน่งเปิดรับ — จำนวนตำแหน่งที่กำลังสรรหาอยู่
  • ผู้สมัคร Active — จำนวนผู้สมัครที่อยู่ในกระบวนการ
  • สัมภาษณ์สัปดาห์นี้ — จำนวนนัดสัมภาษณ์ที่มีกำหนดในสัปดาห์นี้
  • Offer ที่ส่ง — จำนวน Offer ที่ส่งให้ผู้สมัครแล้ว
  • ทดลองงาน — จำนวนพนักงานใหม่ที่อยู่ในช่วงทดลองงาน

4.3 รูปแบบ Layout (3 แบบ)

  1. Standard — กราฟ + ตารางสัมภาษณ์ + Requisition ล่าสุด
  2. Chart Focus — เน้นกราฟ Headcount, Recruitment Trend, Time-to-Hire
  3. Compact — แสดงข้อมูลกระชับสำหรับจอเล็ก

เปลี่ยน Layout ได้โดยคลิกที่ปุ่มเลือก Layout มุมขวาบนของ Dashboard

4.4 กราฟและแผนภูมิ

  • Headcount Trend — กราฟเส้นแสดงแนวโน้มจำนวนพนักงาน 6 เดือน
  • Recruitment Funnel — Funnel Chart แสดงอัตราส่วนผู้สมัครแต่ละขั้นตอน
  • Open vs Hired — กราฟแท่งเปรียบเทียบตำแหน่งเปิด vs จ้างได้
  • Time-to-Hire — กราฟเส้นแสดงจำนวนวันเฉลี่ยในการสรรหา

5. บริหารอัตรากำลัง (Manpower Planning)

5.1 จุดประสงค์

โมดูลนี้ใช้วางแผนและติดตามอัตรากำลังคนของแต่ละแผนก เทียบจำนวนที่วางแผนไว้ (Plan) กับจำนวนจริง (Actual)

5.2 ข้อมูลที่แสดง

  • แผนก — ชื่อแผนก/ฝ่าย
  • Plan — จำนวนอัตราที่วางแผนไว้
  • Actual — จำนวนพนักงานปัจจุบัน
  • Gap — ส่วนต่างระหว่าง Plan กับ Actual (สีแดง = ขาด, สีเขียว = ครบ)
  • แผนภูมิกะ — แสดงตารางกะ A/B/C พร้อมจำนวนคน

5.3 การใช้งาน

  1. เลือกเมนู "อัตรากำลัง" จาก Sidebar
  2. ดูภาพรวมอัตรากำลังแต่ละแผนก
  3. คลิกที่แผนกเพื่อดูรายละเอียดกะ
  4. ผู้ที่มีสิทธิ์ Edit สามารถแก้ไขจำนวน Plan/Actual ได้

6. คำขอเปิดอัตรา (Requisition)

6.1 ภาพรวม

ระบบ Requisition ใช้สำหรับเปิดคำขออัตรากำลังใหม่ พร้อมขั้นตอนการอนุมัติ 4 ระดับ:

  1. ร่าง (Draft) — ผู้ขอจัดทำคำขอ
  2. รอ HR Manager อนุมัติ — ส่งให้ HR ตรวจสอบ
  3. รอกรรมการผู้จัดการอนุมัติ — ส่งให้ MD อนุมัติ
  4. อนุมัติแล้ว / ไม่อนุมัติ — ผลการพิจารณา

6.2 การเปิดคำขอใหม่

1 ไปที่เมนู "คำขอเปิดอัตรา"
2 คลิกปุ่ม "เปิดคำขอใหม่" (มุมขวาบน)
3 กรอกข้อมูล:
  • ตำแหน่งที่ต้องการ (จำเป็น)
  • แผนก
  • จำนวนอัตรา
  • ประเภทพนักงาน (ประจำ / รายวัน / สัญญาจ้าง / ชั่วคราว)
  • สาเหตุ (ทดแทน / ขยายกำลังคน / โครงการใหม่ / ปรับโครงสร้าง)
  • ความเร่งด่วน (สูง / กลาง / ต่ำ)
  • เงินเดือนที่ขอ
  • วันที่ต้องการเริ่มงาน
4 คลิก "ส่งคำขอ" — ระบบจะส่งคำขอให้ HR Manager อนุมัติ

6.3 การอนุมัติคำขอ

ผู้มีสิทธิ์อนุมัติ (HR Manager / MD) จะเห็นปุ่ม:

  • อนุมัติ — อนุมัติและส่งต่อขั้นตอนถัดไป
  • ไม่อนุมัติ — ปฏิเสธคำขอ พร้อมเหตุผล

6.4 สถานะคำขอ

สถานะความหมาย
ร่างยังไม่ได้ส่ง กำลังจัดทำ
รอ HRส่งให้ HR Manager พิจารณาแล้ว
รอ MDHR อนุมัติแล้ว รอกรรมการผู้จัดการ
อนุมัติแล้วได้รับอนุมัติ พร้อมเริ่มสรรหา
ไม่อนุมัติถูกปฏิเสธ
ปิดงานสรรหาครบแล้ว ปิดคำขอ

7. Job Description (JD)

7.1 ภาพรวม

คลัง JD กลางขององค์กร — รวบรวม Job Description ทุกตำแหน่ง รองรับ Version Control เพื่อติดตามการเปลี่ยนแปลง

7.2 การสร้าง JD ใหม่

1 ไปที่เมนู "Job Description"
2 คลิกปุ่ม "สร้าง JD"
3 กรอกข้อมูล:
  • ชื่อตำแหน่ง (จำเป็น)
  • แผนก
  • รายงานต่อ (หัวหน้างาน)
  • สถานะ (ร่าง / รอตรวจ / อนุมัติ)
4 คลิก "สร้าง JD"

7.3 การดู JD

คลิกที่การ์ด JD เพื่อดูรายละเอียด:

  • หน้าที่ความรับผิดชอบ (Responsibilities)
  • คุณสมบัติ (Qualifications)
  • KPI
  • ข้อมูลเวอร์ชัน

7.4 สถานะ JD

สถานะความหมาย
อนุมัติใช้งานได้ เป็นเวอร์ชันปัจจุบัน
รอตรวจกำลังรอตรวจสอบ/อัปเดต
ร่างยังไม่เสร็จ กำลังจัดทำ

8. Pipeline สรรหา (Recruitment Pipeline)

8.1 ภาพรวม

แสดงผู้สมัครทั้งหมดในรูปแบบ Kanban Board แบ่งเป็น 8 สเตจ ผู้ใช้สามารถลากย้าย (Drag & Drop) ผู้สมัครระหว่างสเตจได้

8.2 สเตจ (Stages)

#สเตจคำอธิบาย
1ผู้สมัครใหม่สมัครเข้ามา ยังไม่ได้คัดกรอง
2คัดกรองตรวจสอบคุณสมบัติเบื้องต้น
3สัมภาษณ์ทางโทรศัพท์สัมภาษณ์คัดกรองเบื้องต้น
4สัมภาษณ์ครั้งที่ 1สัมภาษณ์กับหัวหน้างาน
5สัมภาษณ์ครั้งที่ 2สัมภาษณ์เชิงลึก
6สัมภาษณ์ผู้บริหารสัมภาษณ์กับ MD
7เสนอ Offerจัดทำ Offer Letter
8รับเข้าทำงานตอบรับ Offer เริ่มงาน

8.3 การเพิ่มผู้สมัครใหม่

1 คลิกปุ่ม "เพิ่มผู้สมัคร" ที่มุมขวาบน
2 กรอกข้อมูลผู้สมัคร:
  • ชื่อ-นามสกุล (จำเป็น)
  • ตำแหน่งที่สมัคร (จำเป็น)
  • แผนก
  • ช่องทางสมัคร (JobThai / LinkedIn / Walk-in / แนะนำพนักงาน / จัดหางานจังหวัด / Agency)
  • ประสบการณ์ (ปี)
  • เงินเดือนคาดหวัง
  • จังหวัด
  • วุฒิการศึกษา
  • ทักษะ (คั่นด้วยคอมม่า)
3 คลิก "เพิ่มผู้สมัคร" — ระบบจะเพิ่มเข้า Pipeline ที่สเตจ "ผู้สมัครใหม่"

8.4 การย้ายสเตจ (Drag & Drop)

  1. คลิกค้างที่การ์ดผู้สมัคร
  2. ลากไปวางที่คอลัมน์สเตจที่ต้องการ
  3. ระบบจะอัปเดตสถานะอัตโนมัติ

9. ฐานข้อมูลผู้สมัคร (Candidates)

9.1 ภาพรวม

แสดงข้อมูลผู้สมัครทั้งหมดในรูปแบบตาราง พร้อมฟิลเตอร์และค้นหา

9.2 การค้นหาและกรอง

  • ค้นหา — พิมพ์ชื่อ ตำแหน่ง หรือทักษะ ในช่องค้นหา
  • กรองตามแผนก — เลือกแผนกจาก Dropdown
  • กรองตามสเตจ — เลือกสเตจปัจจุบัน

9.3 รายละเอียดผู้สมัคร

คลิกที่แถวผู้สมัครเพื่อดูรายละเอียด:

  • โปรไฟล์ — ข้อมูลส่วนตัว ทักษะ ประวัติการทำงาน
  • AI Matching — คะแนนความเหมาะสมกับ JD, จุดแข็ง/จุดอ่อน, Skill Gap
  • เอกสาร — เอกสารที่แนบมา (Resume, สำเนาบัตร ฯลฯ)

9.4 AI Match Score

ระบบ AI จะให้คะแนนความเหมาะสม (0-100%) โดยพิจารณาจาก:

  • ทักษะที่ตรงกับ JD
  • ประสบการณ์ที่เกี่ยวข้อง
  • วุฒิการศึกษา
  • เงินเดือนคาดหวัง vs งบที่ตั้งไว้
คะแนนระดับสี
85–100%เหมาะสมสูงเขียว
75–84%เหมาะสมดีน้ำเงิน
65–74%เหมาะสมปานกลางเหลือง
ต่ำกว่า 65%ไม่ค่อยเหมาะแดง

10. การสัมภาษณ์ (Interview)

10.1 ภาพรวม

โมดูลจัดการนัดสัมภาษณ์ ตารางนัดหมาย และแบบประเมินคะแนนมาตรฐาน 6 หัวข้อ

10.2 การสร้างนัดสัมภาษณ์ใหม่

1 คลิกปุ่ม "สร้างนัด"
2 กรอกข้อมูล:
  • ชื่อผู้สมัคร (จำเป็น)
  • ตำแหน่ง
  • ประเภท (Onsite / Online)
  • รอบสัมภาษณ์ (ครั้งที่ 1 / ครั้งที่ 2 / ผู้บริหาร)
  • วันที่ (จำเป็น)
  • เวลา (จำเป็น)
  • กรรมการสัมภาษณ์ (คั่นด้วยคอมม่า)
3 คลิก "สร้างนัด"

10.3 การประเมินผลสัมภาษณ์

1 ที่รายการนัดสัมภาษณ์ คลิกปุ่ม "ประเมิน"
2 ให้คะแนน 1–5 ใน 6 หัวข้อ:
  • ทักษะเฉพาะตำแหน่ง (Technical)
  • ประสบการณ์ (Experience)
  • บุคลิกภาพ / ทัศนคติ (Personality)
  • ภาวะผู้นำ (Leadership)
  • การสื่อสาร (Communication)
  • การแก้ปัญหา (Problem Solving)
3 คลิก "บันทึกผลประเมิน" — ระบบคำนวณคะแนนเฉลี่ยอัตโนมัติ

10.4 สถานะสัมภาษณ์

สถานะความหมาย
Scheduledมีนัดหมาย รอสัมภาษณ์
Doneสัมภาษณ์แล้ว มีคะแนนประเมิน

11. Offer Management

11.1 ภาพรวม

จัดทำ Offer Letter พร้อมโครงสร้างค่าตอบแทน และขั้นตอนอนุมัติก่อนส่งผู้สมัคร

11.2 ขั้นตอนอนุมัติ Offer

  1. HR จัดทำ — เจ้าหน้าที่สรรหาสร้าง Offer
  2. กรรมการผู้จัดการอนุมัติ — MD พิจารณาอนุมัติ
  3. ส่งผู้สมัคร — ส่ง Offer Letter ทาง Email
  4. ตอบรับ — ผู้สมัครตอบรับ/ปฏิเสธ

11.3 การสร้าง Offer ใหม่

1 คลิกปุ่ม "สร้าง Offer"
2 กรอกโครงสร้างค่าตอบแทน:
  • ชื่อผู้สมัคร (จำเป็น)
  • ตำแหน่ง (จำเป็น)
  • เงินเดือนพื้นฐาน (฿)
  • ค่าตำแหน่ง (฿)
  • ค่าเดินทาง (฿)
  • เบี้ยขยัน (฿)
  • โบนัส (เช่น "2 เดือน/ปี")
  • กองทุนสำรองเลี้ยงชีพ (เช่น "5%")
3 คลิก "สร้าง Offer"

11.4 การอนุมัติ Offer (MD)

กรรมการผู้จัดการ (MD) จะเห็นปุ่ม:

  • อนุมัติ & ส่งให้ผู้สมัคร — อนุมัติพร้อมส่ง Email ถึงผู้สมัคร
  • ตีกลับ — ส่งกลับให้ HR แก้ไข

11.5 สถานะ Offer

สถานะความหมาย
ร่างกำลังจัดทำ
รออนุมัติ (MD)ส่งให้ MD พิจารณาแล้ว
ส่งให้ผู้สมัครแล้วส่ง Offer Letter แล้ว รอตอบ
ตอบรับแล้วผู้สมัครตอบรับ Offer
ปฏิเสธผู้สมัครปฏิเสธ Offer

12. Onboarding

12.1 ภาพรวม

ติดตามกระบวนการ Onboard พนักงานใหม่ ตั้งแต่เตรียมเอกสาร อุปกรณ์ ระบบ จนถึงอบรม

12.2 การใช้งาน

  1. เลือกพนักงานใหม่จากรายการด้านซ้าย
  2. ดู Checklist งานที่ต้องดำเนินการ
  3. คลิกที่ checkbox เพื่อทำเครื่องหมายว่าเสร็จแล้ว
  4. แถบ Progress Bar จะอัปเดตอัตโนมัติ

12.3 กลุ่มงาน Onboarding

กลุ่มตัวอย่างงาน
เอกสารจัดเก็บสำเนาบัตร/ทะเบียนบ้าน/วุฒิ, ตรวจสุขภาพ
อุปกรณ์ชุดยูนิฟอร์ม, รองเท้าเซฟตี้, บัตรพนักงาน, คีย์การ์ด
ระบบสร้าง Email บริษัท, สร้าง User Account (ERP/HRIS)
อบรมSafety Induction, ปฐมนิเทศ (Orientation)

13. รายงาน (Reports)

13.1 รายงานที่มีให้

รายงานคำอธิบาย
Open Position Reportตำแหน่งที่เปิดรับทั้งหมดและสถานะปัจจุบัน
Candidate Status Reportสถานะผู้สมัครทั้งหมดใน Pipeline
Interview Result Reportสรุปผลการสัมภาษณ์ คะแนน ผู้ผ่าน/ไม่ผ่าน
Hiring Reportสถิติการจ้างงานแยกตามแผนก/ช่วงเวลา
Turnover Reportอัตราการลาออกแต่ละแผนก
Probation Reportผลประเมินช่วงทดลองงาน

13.2 การส่งออกรายงาน

  1. เลือกช่วงเวลา (ไตรมาสนี้ / เดือนนี้ / ปีนี้ / กำหนดเอง)
  2. คลิกปุ่ม "Excel" หรือ "PDF" ที่รายงานที่ต้องการ
  3. ระบบจะดาวน์โหลดไฟล์อัตโนมัติ

14. AI Analytics

14.1 ภาพรวม

ระบบ AI วิเคราะห์ข้อมูลการสรรหาเชิงลึก ให้ข้อมูลเชิงกลยุทธ์เพื่อปรับปรุงกระบวนการ

14.2 ความสามารถ AI

  • Resume Screening — คัดกรอง Resume อัตโนมัติเทียบกับ JD
  • AI Matching Score — ให้คะแนนความเหมาะสมผู้สมัครกับตำแหน่ง
  • Interview Summary — สรุปจุดเด่น/จุดด้อยจากความเห็นกรรมการ
  • Recruitment Insights — วิเคราะห์แนวโน้มและข้อเสนอแนะ

14.3 ตัวอย่าง AI Insights

  • ตำแหน่งที่หายาก — ระบบแจ้งเตือนตำแหน่งที่ใช้เวลาสรรหานาน พร้อมแนะนำช่องทางเพิ่มเติม
  • สาเหตุปฏิเสธ Offer — วิเคราะห์เหตุผลที่ผู้สมัครปฏิเสธ แนะนำสวัสดิการเสริม
  • แนวโน้มลาออก — ตรวจจับอัตราออกช่วงทดลองงานสูงผิดปกติ
  • ประสิทธิภาพช่องทาง — เปรียบเทียบ Quality of Hire แต่ละช่องทาง

15. การแจ้งเตือน (Notifications)

15.1 ภาพรวม

ศูนย์รวมการแจ้งเตือนทั้งหมด สามารถกำหนดช่องทางและเหตุการณ์ที่ต้องการรับแจ้งเตือนได้

15.2 ช่องทางแจ้งเตือน

ช่องทางคำอธิบายการตั้งค่า
LINE Notifyแจ้งเตือนผ่าน LINE กลุ่มเปิด/ปิดได้
LINE OA APIแจ้งเตือนผ่าน LINE Official Accountเปิด/ปิดได้
Email (SMTP)ส่งอีเมลแจ้งเตือนเปิด/ปิดได้

15.3 การเปิด/ปิดช่องทาง

  1. ไปที่เมนู "การแจ้งเตือน"
  2. ที่แผง "ช่องทางแจ้งเตือน" (ด้านขวา)
  3. คลิกที่สวิตช์ของแต่ละช่องทางเพื่อเปิด (สีเขียว) หรือปิด (สีเทา)

15.4 เหตุการณ์ที่แจ้งเตือน

สามารถเลือกเปิด/ปิดการแจ้งเตือนแต่ละเหตุการณ์:

  • อนุมัติเปิดอัตรา
  • นัดสัมภาษณ์
  • แจ้งผลสัมภาษณ์
  • ส่ง Offer
  • เตือนเอกสารขาด

คลิกที่ checkbox แต่ละเหตุการณ์เพื่อเปิด/ปิด (เครื่องหมายถูก = เปิดรับ, ขีดทับ = ปิด)

15.5 การอ่านแจ้งเตือน

  • แจ้งเตือนที่ยังไม่อ่าน จะมีจุดสีน้ำเงินและพื้นหลังเน้น
  • คลิก "ทำเครื่องหมายว่าอ่านแล้ว" เพื่อทำเครื่องหมายทั้งหมด

17. การจัดการผู้ใช้

17.1 ผู้ที่จัดการได้

เฉพาะ Super Admin เท่านั้น

17.2 การเพิ่มผู้ใช้ใหม่

1 ไปที่ ตั้งค่า → จัดการผู้ใช้
2 คลิก "เพิ่มผู้ใช้"
3 กรอกข้อมูล:
  • ชื่อผู้ใช้ (Username)
  • ชื่อที่แสดง (Display Name)
  • บทบาท (Role)
  • รหัสผ่าน (ขั้นต่ำ 8 ตัว)
4 คลิก "บันทึก"

17.3 การแก้ไขผู้ใช้

  1. คลิกปุ่ม "แก้ไข" ที่แถวผู้ใช้
  2. แก้ไขชื่อที่แสดง, บทบาท, หรือสถานะ (Active/Inactive)
  3. คลิก "บันทึก"

17.4 การรีเซ็ตรหัสผ่าน

  1. คลิกปุ่ม "รีเซ็ตรหัสผ่าน" ที่แถวผู้ใช้
  2. กรอกรหัสผ่านใหม่ (ขั้นต่ำ 8 ตัว)
  3. คลิก "รีเซ็ต"
  4. แจ้งรหัสผ่านใหม่ให้ผู้ใช้

17.5 การปิดการใช้งาน (Deactivate)

สามารถปิดการใช้งานผู้ใช้ได้โดยเปลี่ยนสถานะเป็น "Inactive" — ผู้ใช้จะไม่สามารถเข้าสู่ระบบได้
